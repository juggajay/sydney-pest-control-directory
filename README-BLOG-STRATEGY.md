## BLOG FORMULA

Every blog post should include:

```
Header Structure:
- H1: Primary keyword (1 per post)
- H2: Secondary keywords (2-5 per post)
- H3: LSI keywords (2-3 per H2)

Internal Links (10-20 per post):
- Service pages: 2-4
- Suburb pages: 3-8
- Blog posts: 3-5
- CTA links: 1-2

Schema Markup:
- BlogPosting (required)
- FAQ (if 5+ FAQs)
- HowTo (if step-by-step)
- BreadcrumbList (required)

Content:
- 1,200-3,500 words
- Table of Contents
- Multiple H2 sections
- Lists/tables for scanability
- 0-5 featured snippet targets
- 1-2 CTA sections

Meta Tags:
- Title: 50-60 characters
- Description: 150-160 characters
- OG tags for social
```

## COMMON QUESTIONS

**Q: Which document should I read first?**
A: Start with BLOG-STRATEGY-SUMMARY.md. It explains all documents and helps you decide what to read based on your role.

**Q: Can I copy the React component?**

**Q: What's the recommended reading order?**
A: Summary → Strategy → Examples → Quick Reference (as needed)

**Q: How often should I update posts?**
A: Update top posts monthly, others quarterly. Refresh stats, links, and modification date.

**Q: Should I publish seasonal content in advance?**
A: Yes. Publish 3 weeks before season starts (mid-August for spring).
